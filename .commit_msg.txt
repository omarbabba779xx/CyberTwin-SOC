API refactor (start of progressive split of the 1591-line main.py):
- New backend/api/deps.py: shared singletons (limiter, orchestrator) and
  helpers (_client_ip, _get_cached_result, _safe_path) so routers can
  re-use them without re-importing main.
- New backend/api/routes/ package with 4 cleanly-extracted routers:
    - health.py       (/, /api/health, /api/health/deep, /api/metrics)
    - auth.py         (/api/auth/login, /api/auth/me, /api/audit)
    - environment.py  (/api/environment[/hosts|/users])
    - history.py      (/api/history/*, 5 endpoints)
- main.py: imports limiter+orchestrator from deps, calls
  app.include_router(...) for the 4 routers, removes the migrated
  inline endpoint definitions.
- Net effect on main.py: 1591 -> 1406 lines (-185, -12%).
- 13 of the legacy ~75 endpoints now live in their domain router.
- Pattern is now established; the remaining groups (scenarios,
  simulation, results, mitre, sigma, coverage, alerts, cases,
  suppressions, connectors, ingest, soar, threat-intel) can migrate
  one PR at a time without touching main.py more than to remove blocks
  and add include_router calls.
